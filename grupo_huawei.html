Desctripción del Tema principal del producto

Los Alumnos del Campus Fia de la Universidad San Martin de Porres, han tenido ciertos problemas con la información de la misma universidad, como son los horarios, matriculas, capacitaciones, conferencias, webinar, cancelación de clases, áreas donde se está haciendo la limpieza, si está libre el estacionamiento, registro de asistencias, Ubicación GPS, lugares más visitados y donde pueda haber una mejora de implementación.
Crear un sitio web Responsive, que pueda tener todas las áreas del Campus Fia USMP, y tenga el acceso de poder incluir un chatbot el cual nos permita informar a todos los alumnos sobre que acontecimientos se realizarán, y que también se pueda registrar todas las asistencias de cada alumno, el chatbot podrá direccionar al alumno al área con la que se quiera comunicar y también se podrá crear un chat en línea donde todos los alumnos puedan comunicarse.
Alumnos más informados, eventos con aforos satisfactorios, mejora en la implementación de las áreas más visitadas por los alumnos, buena información de matrículas y horarios, registro de asistencias satisfactorio.

Arquitectura

Chat Campus Fia SC AWS

Para desarrollar nuestro sitio Web elaboramos una arquitectura con los siguientes servicios que ofrece AWS.

Arquitectura Versión 4.0.0

Chat Campus Fia SC AWS

Implementaremos una web Responsive donde todos los alumnos puedan atraves de un chat comunicarse en diferentes áreas y así estar enterados de acontecimientos, eventos que se den en la universidad, también implementar un GPS para cada alumno y así poder registrarlos y ponerles asistencia utilizando un sensor que sea reconocido por un celular. Comenzamos nuestra arquitectura con el usuario que ingresa al CHAT CAMPUS mediante la web en su ordenador, este chat recibe la información mediante el API Gateway, componente cuya principal función es la de habilitar la conexión entre los servicios y los clientes, trabajando en su back-end mediante Amazon S3, GitHub y Visual Studio Code. Luego seguimos con Amazon Lex, que es un chat bot donde los usuarios (estudiantes) podrán realizar sus preguntas para acceder a los diferentes pabellones, ubicación (GPS) e información específica, el agente virtual de texto les va a ofrecer una experiencia instantánea para los usuarios ya que estos necesitan respuestas rápidas y precisas a cualquier hora del día y puedan solucionar problemas o encontrar la información que estaban buscando con urgencia, y que también estará enlazado con el API Gateway que enviara transformara la información para que se comprenda y con AWS Lambda que se encargar de hacerle llegar la información que el GPS emita acompañado de una integración que se realiza de manera sencilla por ser las 2 plataformas de AWS mediante unas llaves o ‘keys’ que se obtienen en la configuración. Y con AWS Lambda podremos ejecutar los códigos provenientes de Amazon Lex y ser administrados automáticamente, ejecutando la lógica front-end en el lado del cliente y los servicios de seguridad, la BD y la lógica de back-end, esto con la idea de ahorrarnos tiempo y que nos permita desarrollar el chatbot. También se implementará Amazon Location Services que enviará la ubicación del usuario, integrándose mediante AWS Lambda con la misma modalidad de llaves o ‘keys’, para la ubicación de todos los usuarios que se encuentren en Fia, a través de un sensor GPS que está incluido en cada Smartphone.
